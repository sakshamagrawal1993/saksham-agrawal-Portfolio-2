**From Cognitive Skills to Packaged Skills**: An important evolution in 2025–2026 is the emergence of the **Agent Skills open standard** (`agentskills.io`), originally developed by Anthropic. While the taxonomy above describes *what cognitive capabilities* an agent needs, the Agent Skills standard defines *how procedural knowledge is packaged and delivered*. A Skill is a portable, filesystem-based package (`SKILL.md` file + optional scripts, references, and assets) that uses **progressive disclosure** to load instructions on-demand — only the skill name and description load at startup (~30–50 tokens), the full instructions load only when the task matches, and reference documents load only when explicitly needed during execution. This is context engineering at its most disciplined: an agent with 20 skills available pays the context cost of only the one it activates. Skills are the **playbook** (how to reason about a task); MCP tools are the **equipment** (how to execute actions). The `allowed-tools` field in each skill's YAML frontmatter explicitly declares which MCP tools the skill expects, enabling the harness to validate tool availability before activation. For a complete treatment of the Agent Skills standard, progressive disclosure patterns, and healthcare-specific skill design, see [[Anatomy of an AI Agent — The Harness Architecture]].
